# Steps 13 & 14 — Signature Management + User Output

## What Was Done

- Signature `add` and `remove` are now atomic: signature change + version increment in one SQLite transaction.
- Duplicate signatures return a clear error.
- Invalid remove ID returns a clear error.
- Added `excluded` counter to scan summary and DB session.

### Files Modified

```
src/database/SignatureRepository.cpp      (transaction, duplicate/invalid-id errors)
src/signatures/SignatureService.cpp       (removed redundant incrementVersion calls)
include/database/ScanSessionRepository.h (added excluded param to updateCounters)
src/database/ScanSessionRepository.cpp   (excluded_files column in UPDATE)
include/scan/ScanService.h               (added excluded to Counters)
src/scan/ScanService.cpp                 (count exclusions, pass to summary + DB)
```

## Test Instructions

### 1 — Setup

```bash
./build.sh
rm -f scanner-data/scanner.db
```

---

### 2 — Add signature

```bash
./build/scanner signatures add "evil_payload"
# → Signature added with id 1.
```

---

### 3 — Duplicate rejected

```bash
./build/scanner signatures add "evil_payload"
# → Error: Signature already exists: evil_payload
```

---

### 4 — Empty value rejected

```bash
./build/scanner signatures add ""
# → Error: Signature value cannot be empty.
```

---

### 5 — List

```bash
./build/scanner signatures list
# ID  Value
# 1   evil_payload
```

---

### 6 — Remove invalid ID

```bash
./build/scanner signatures remove 99
# → Error: Signature not found: 99
```

---

### 7 — Remove valid ID and verify version increments

```bash
./build/scanner signatures add "second_sig"
# → Signature added with id 2.

sqlite3 scanner-data/scanner.db \
  "SELECT key, value FROM scanner_metadata WHERE key='signature_version';"
# → signature_version|3

./build/scanner signatures remove 2
# → Signature 2 removed.

sqlite3 scanner-data/scanner.db \
  "SELECT key, value FROM scanner_metadata WHERE key='signature_version';"
# → signature_version|4
```

---

### 8 — Cache invalidation after signature change

```bash
mkdir -p test-data
echo "clean content" > test-data/file.txt

./build/scanner scan --path test-data
# Files scanned: 1, Cache hits: 0

./build/scanner scan --path test-data
# Files scanned: 0, Cache hits: 1   ← cache hit

./build/scanner signatures add "clean"
./build/scanner scan --path test-data
# Files scanned: 1, Cache hits: 0   ← version bump invalidated cache
```
